# Snapshot: สรุปการพูดคุยล่าสุด - Auth UX (เลือกวิธีการแก้ไข)

**เวลา**: 2025-12-23 23:45 GMT+7
**บริบท**: สรุปตัวเลือกสำหรับการแก้ปัญหา Authentication UX ที่หน้า Home (ผู้ใช้ยังไม่ได้เลือก A/B/C)

## สรุปการตัดสินใจล่าสุด
- เราเห็นปัญหา: หน้า Home อนุญาตให้ส่งคำถามโดยไม่ล็อกอิน แต่ API ต้องการ session → ผู้ใช้ที่ไม่ล็อกอินจะเจอ 401 และ UX ที่สับสน
- ผมเสนอ 3 ตัวเลือกการแก้ไขที่หน้า Home:
  - **A**: แทนที่ input ด้วยปุ่ม **LINE Login** (เรียบง่าย ป้องกันการส่ง)
  - **B**: เก็บ input ไว้ แต่ทำเป็น **disabled + Login CTA** (ผู้ใช้เห็น input แต่ต้องล็อกอินก่อนส่ง)
  - **C**: แบบ B + **auto-submit หลังล็อกอิน** (ถ้าผู้ใช้พิมพ์ไว้และล็อกอินเรียบร้อยแล้ว จะส่งคำถามให้โดยอัตโนมัติ)

## สถานะตอนนี้
- **ผู้ใช้ยังไม่ได้เลือก A/B/C** — การดำเนินการจริงยังไม่เริ่ม (Step A รอการตัดสินใจ)

## Next step เมื่อผู้ใช้เลือก
- เริ่มแก้ตามตัวเลือกที่เลือก (แก้ `app/page.tsx`, เพิ่ม modal/CTA, เพิ่ม unit tests และ manual test)
